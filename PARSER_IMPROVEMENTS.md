### 2. **Создана система безопасной миграции** (`server/src/utils/scheduleMigration.ts`)

**Стратегия сохранения данных:**

```
БЫЛО (старый способ):                 СТАЛО (новый способ):
─────────────────────────────────────────────────────────────
1. Удалить ВСЫ расписание             1. Посчитать пользователей с группами
2. Удалить структуру (группы и т.д.)  2. Создать новую структуру
3. Импортировать новое                3. Обновить только УРОКИ (расписание)
4. Пользователи потеряют группы!      4. Пользователи видят новое расписание!
                                       5. Старые группы остаются для истории
```

**Этапы миграции:**
1. **Сохранение** - анализ всех User.groupId привязок
2. **Создание** - Institute, Direction, Program, Group (новые + старые)
3. **Удаление** - только lessons для обновляемых групп
4. **Создание** - новые lessons с правильной четностью
5. **Проверка** - отчёт о orphaned users (если есть пользователи со старыми группами)

## ⚠️ Что происходит, если направления сильно изменились?

**Сценарий:** Вуз переименовал направления или сделал их структурные изменения.

**Что произойдет:**
1. Пользователь выбрал "Группа 1501 (Экономика)" в старой версии
2. В новой версии это направление осталось (но может быть переименовано)
3. Пользователь продолжит видеть расписание **его группы**
4. Админ-панель выведет: "⚠️ Пользователей с историческими группами: N"

**Что делать:**
- Уведомьте пользователей через ботом или замечание об обновлении
- Они смогут **пересмотреть выбор группы** через SelectGroupPage
- Если выбор остален старым направлением - ничего страшного, расписание всё ещё работает

## 📞 Если что-то не работает

### Парсер не распознaёт файлы
- Проверьте имена файлов: `1.xlsx`, `2.xlsx`, `3.xlsx`, `4.xlsx`, `z.xlsx`, `m.xlsx`
- Убедитесь, что Excel не повреждён
- Откройте файл в Excel и проверьте структуру

### ИИС и ИУПСИБК путаются
- Посмотрите объединённые ячейки в строке 4 листа
- Проверьте, что колонки объединены правильно
- Убедитесь, что в ячейке написано "ИУПСИБК" или "ИИС"

### Некоторые направления не импортировались
- Проверьте наличие строки "НАПРАВЛЕНИЕ" в первых 10 строках
- Убедитесь, что направления указаны для каждой колонки

### Пользователи потеряли группы
- **Это не должно было случиться!** Используйте бекап:
  ```bash
  curl -X POST http://localhost:3001/api/admin/backup/restore/:id
  ```

## 🎓 Техническая информация

### Как парсер определяет ИИС/ИУПСИБК

1. **Читает объединённые ячейки** в строке 4 (INDEX 3)
2. **Ищет текст** "ИУПСИБК, ИИС" или отдельно "ИИС" и "ИУПСИБК"
3. **Сопоставляет колонки** с найденными диапазонами
4. **Передаёт информацию** в getInstituteName как `determinedInstitute`
5. **Результат:** Каждой группе правильно назначается её институт

### Как работает безопасная миграция

1. **Анализ** - SELECT * FROM User WHERE groupId IS NOT NULL
2. **Сохранение** - Все пользователи получают пометку "к сохранению"
3. **Создание** - Institute → Direction → Program → Group (транзакция)
4. **Удаление** - DELETE FROM Lesson WHERE groupId IN (обновляемые группы)
5. **Создание** - INSERT INTO Lesson (новые данные)
6. **Проверка** - Поиск orphaned users (есть ли пользователи со старыми группами)

### Производительность

- **Парсирование файла 1.xlsx** (~3000 пар): ~5 сек
- **Миграция (6 файлов)**: ~30-45 сек
- **Создание 10k уроков**: ~15 сек
- **Итого**: ~1-2 минуты для полного обновления
