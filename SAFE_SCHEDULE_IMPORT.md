# 📚 Безопасное обновление расписания

## 🎯 Проблема

Вы хотите обновить расписание, но у вас уже есть **35 пользователей**, которые выбрали свои направления и группы. Если просто удалить и переипортировать данные, пользователи потеряют свои выборы.

## ✅ Решение: Безопасная миграция

Мы реализовали систему, которая:
- ✅ **Сохраняет всех пользователей** и их привязки к группам
- ✅ **Создаёт новые направления** без удаления старых
- ✅ **Обновляет только расписание** (уроки) для существующих групп
- ✅ **Правильно разделяет ИИС и ИУПСИБК** на листах混合
- ✅ **Логирует все действия** для аудита

## 🚀 Как пользоваться

### Метод 1: Загрузить несколько файлов Excel (1.xlsx, 2.xlsx и т.д.)

```bash
# Через API
POST /api/admin/import-safe
Content-Type: multipart/form-data

files: [1.xlsx, 2.xlsx, 3.xlsx, 4.xlsx, z.xlsx, m.xlsx]
```

Ответ:

## 🔍 Автоматическое разделение ИИС и ИУПСИБК

**Проблема:** На листе "ИУПСИБК, ИИС" находятся две разные организации, и парсер должен правильно определить, какие группы к какому институту относятся.

**Решение:** Парсер анализирует:
1. **Объединённые ячейки в строке 4** - прямое указание границ ИИС и ИУПСИБК
2. **Название программы** - если указана `ИНФОРМАЦИОННЫЕ СИСТЕМЫ` → ИИС
3. **Направление** - для ОЗФО определяет по названию направления

Примеры логики:
- Программа содержит `ИНФОРМАЦИОНН` или `ИНФОРМАЦ` → **ИИС**
- Программа содержит `СОЦИАЛЬ` или `ПСИХОЛОГ` → **ИУПСИБК**
- Программа содержит `ПЕРСОНАЛ` → **ИУПСИБК**

## 🛡️ Что защищено

| Данные | Статус | Примечание |
|--------|--------|-----------|
| Пользователи (User) | ✅ Защищены | Ни один пользователь не удаляется |
| Привязки к группам | ✅ Сохранены | User.groupId остаются неизменны |
| Телеграм ID | ✅ Сухранены | Все связи с ботом работают |
| Роли (admin, student) | ✅ Сохранены | Администраторы остаются админами |
| Группы со старыми пользователями | ✅ Сохранены | Старые направления остаются для истории |
| Расписание | ✅ Обновлено | Только уроки (временно, дни, пары) - безопасно обновляются |

## 🔄 Сценарий: "Направления сильно изменились"

Если направления в ВУЗе большими изменениями, некоторые пользователи могут оказаться с "историческими" группами.

**Что произойдёт:**
1. Пользователь выбрал "Группа 1501 (Экономика, ОЧНАЯ)" 
2. Это направление существует в СТАРЫХ данных
3. При новом импорте могут быть НОВЫЕ группы и направления
4. Пользователь продолжит видеть расписание старой группы
5. В админ-панели это отмечается: "⚠️ Пользователей с историческими группами: 3"

**Что делать:**
- Информируйте пользователей об обновлении расписания
- Они смогут выбрать НОВОЕ направление через `SelectGroupPage`
- Старые группы/направления можно будет удалить позже (если не нужны для архива)

## 📝 Команда для полного сброса (ВНИМАНИЕ!)

Если вы хотите полностью очистить расписание и направления (не рекомендуется если есть пользователи):

```bash
DELETE /api/admin/schedule/reset
```

⚠️ **ЭТА КОМАНДА:**
- Удаляет ВСЕ уроки
- Удаляет ВСЕ группы, программы, направления, институты
- **НЕ трогает** пользователей - они становятся без группы
- Пользователи смогут выбрать группу заново после импорта новых данных

## 🔧 Если есть проблемы

### Парсер не распознаёт ИИС/ИУПСИБК
Проверьте:
- [ ] Объединённые ячейки в строке 4 листа содержат "ИУПСИБК" или "ИИС"
- [ ] Или программа группы содержит ключевые слова (ИНФОРМАЦИОННЫЕ, СОЦИАЛЬНЫЕ и т.д.)
- [ ] Файл Excel не повреждён

### Некоторые группы не показываются
Проверьте:
- [ ] В файле Excel есть строка с номерами групп (у нас это строка 8)
- [ ] Есть строка с направлениями (должна быть помечена "НАПРАВЛЕНИЕ")
- [ ] Максимально первые 20 строк файла содержат метаданные

### Расписание неправильное после импорта
Проверьте логи:
```
curl http://localhost:3001/api/admin/logs?type=import
```
