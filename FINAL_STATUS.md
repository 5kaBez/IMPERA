## 💾 МОДИФИЦИРОВАННЫЕ ФАЙЛЫ

### 1. **guuScheduleImporter.ts** (Авто-импорт)
```typescript
// БЫЛО:
const result = await parseExcelSchedule(xlsxBuffer, prisma);

// СТАЛО:
const migrationStats = await migrateSchedule(groupsData, lessonsData);
```
✅ Теперь вызывает безопасную миграцию вместо простого парсера

### 2. **scheduleMigration.ts** (Новый файл)
Функция `migrateSchedule()` - главный алгоритм:
- Анализирует текущих пользователей
- Создаёт новую структуру
- Обновляет только lessons
- Возвращает статистику

### 3. **guuParser.ts** (Исправленный парсер)
Добавлены:
- Анализ объединённых ячеек (merged cells)
- Правильное определение ИИС vs ИУПСИБК
- MergeInfo тип данных для метаданных
- Fallback логика если merged cells недоступны

### 4. **admin.ts** (API маршруты)
Два endpoint:
- `/api/admin/auto-import` - кнопка в админ-панели (ИСПОЛЬЗУЕТСЯ)
- `/api/admin/import-safe` - резервный ручной endpoint

### Шаг 2: Парсирование
```
Excel файлы → parseGUUScheduleFiles()
├─ Читаем строку 4 (Institute names с объединением ячеек)
├─ Определяем: ИИС или ИУПСИБК
├─ Парсим: Direction → Program → Group → Lesson

```

├─ ОБНОВЛЕНИЕ: Обновляем только Lessons
└─ ИТОГ: Все пользователи видят новое расписание ✅
```

### Шаг 4: Результаты
```

   - Сколько уроков создано
   - Какие новые направления добавлены

## 🆘 TROUBLESHOOTING

### Авто-импорт не работает
```
Проверить:
1. Интернет соединение (нужен доступ к guu.ru)
2. Логи Сервера: docker logs impera-server
3. База данных доступна
4. Админ имеет нужные права
```

### ИИС/ИУПСИБК неправильно разделены
```
Проверить:
1. Объединённые ячейки в Excel (строка 4)
2. Текст в ячейках: "ИНФОРМАЦИОННЫХ СИСТЕМ" → ИИС
3. Если ошибка в файле GUU - восстановить из бекапа
```

### Пользователи потеряны (не должно быть!)
```
Восстановление:
1. Admin → Резервные копии
2. Выбрать бекап ДО импорта
3. Нажать "Восстановить"
4. Все 35 пользователей вернутся
```
